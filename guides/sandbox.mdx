The Ramp sandbox is a testing environment where developers and partners can safely build, test, and experiment with Ramp's API without affecting live production data.

**Key features:**
- Safe testing environment — test integrations before deploying to production
- Full API access — use all Ramp Developer API endpoints
- No real money movement — demo accounts cannot process actual financial transactions

## Access

| Who | How to get access |
|---|---|
| **Developers / Partners** | Submit a request through the [Ramp technology partnerships page](https://ramp.com/partners) |
| **Existing customers** | Request a sales sandbox through your Account Manager |

## Environments

| Environment | URL |
|---|---|
| **Ramp UI (sandbox)** | `https://demo.ramp.com` |
| **API base URL** | `https://demo-api.ramp.com` |

All API calls in the sandbox should use `https://demo-api.ramp.com` as the base URL instead of `https://api.ramp.com`.

## Demo Actions

The sandbox supports simulation actions for testing. Open the demo actions panel with `⌘ J` from any page in the Ramp sandbox UI.

### Mark a Bill as Paid
Navigate to **Bill Pay**, open the bill, press `⌘ J`, and select **Pay current bill**.

<Note>
  This action depends on your Bill Pay payment release setting. When enabled, it only works for bills that are approved and scheduled. When disabled, it works for all bills.
</Note>

### Mark a Reimbursement as Paid
Navigate to the **Reimbursements** tab → **Needs review**, open the reimbursement, press `⌘ J`, and select **Mark reimbursement as paid**.

### Create a Transaction
From any page, press `⌘ J` and click **Add transactions**.

## Demo Action Permissions

| Demo Action | Required Role |
|---|---|
| Pay current bill | Admin, Business Owner, or AP Clerk permission |
| Mark reimbursement as approved | Direct manager, Admin, or Bookkeeper |
| Create a transaction | Employee role or higher (Manager, Admin, Bookkeeper) |

<Note>
  View-only Admins and Guests cannot create transactions.
</Note>
