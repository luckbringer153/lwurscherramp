title: "Tax Codes"
description: "Associate tax rates with transactions in Ramp to keep your ERP in sync on tax classification."
---

<Warning>
  Tax code endpoints are currently in Beta. During the beta, endpoints may be subject to breaking changes, which will be announced via the [Changelog](/changelog).
</Warning>

Tax codes allow you to associate tax rates with transactions in Ramp, keeping your ERP and Ramp in sync on tax classification.

**Prerequisites:**
- An active accounting connection
- GL accounts already uploaded (required for associating tax rates with GL accounts)
- `accounting:read` and `accounting:write` scopes

## Setup (3 Steps in Order)

<Steps>
  <Step title="Upload Tax Rates">
    Tax rates define the individual rate values (e.g., GST at 5%, PST at 7%) that will be available as options within a tax code.

    ```bash
    POST /developer/v1/accounting/tax/rates
    {
      "tax_rates": [
        {
          "id": "gst-5",
          "name": "GST 5%",
          "rate": "0.05",
          "accounting_gl_account_id": "GL_RAMP_ID_HERE"
        },
        {
          "id": "pst-7",
          "name": "PST 7%",
          "rate": "0.07",
          "accounting_gl_account_id": "GL_RAMP_ID_HERE"
        }
      ]
    }
    ```

    | Field | Description |
    |---|---|
    | `id` | Your ERP's external identifier for this tax rate |
    | `name` | Display name of the tax rate |
    | `rate` | Decimal value (e.g., `"0.05"` for 5%) |
    | `accounting_gl_account_id` | The `ramp_id` of the GL account to associate with this rate. Use `GET /developer/v1/accounting/accounts` to find it. |

    You can upload up to 500 tax rates per request. Batches are all-or-nothing.
  </Step>
  <Step title="Create the Tax Code Field">
    The tax code field is the accounting field that holds your tax code options. There can only be one active tax code field per accounting connection — only do this once.

    ```bash
    POST /developer/v1/accounting/tax/code
    {
      "name": "Tax Code",
      "display_name": "Tax Code"
    }
    ```

    If the tax code field already exists, skip to Step 3.
  </Step>
  <Step title="Upload Tax Code Options">
    Tax code options are the selectable values within your tax code field. Each option can reference one or more tax rates from Step 1.

    ```bash
    POST /developer/v1/accounting/tax/code/options
    {
      "options": [
        {
          "id": "hst-on",
          "name": "HST (Ontario)",
          "tax_rate_ids": ["gst-5", "pst-7"]
        }
      ]
    }
    ```

    You can upload up to 500 options per request.
  </Step>
</Steps>

## In the Ramp UI

Once tax code options are uploaded, a **Tax Code** field appears to end users as a selectable coding field on transactions.

## In the API

After a user selects a tax code, it appears in the `accounting_field_selections` array for line items when fetching transactions:

```json
{
  "accounting_field_selections": [
    {
      "external_id": "hst-on",
      "name": "HST (Ontario)",
      "type": "TAX_CODE",
      "category_info": {
        "type": "TAX_CODE"
      }
    }
  ]
}
```

The `"type": "TAX_CODE"` identifier makes it easy to find tax code selections programmatically.
