A bill is the structured accounts payable record in Ramp — the object that powers approvals, coding, and payment.

**Common use cases:**
- **ERP integration** — Sync bills and payments between Ramp and your accounting system
- **Bill processing automation** — Programmatically create bills that are automatically approved for payment
- **Financial reporting** — Extract bill and payment data for custom reporting and analytics

## How Bills Work

Payments are nested within bills and do not have their own dedicated endpoints. When you create a bill via API, you include payment details in the same request. Payment information is returned as part of the bill object on `GET /bills`.

### Bill Lifecycle

| Status | Description |
|---|---|
| `Draft` | Invoice uploaded, OCR extracts details |
| `Pending approval` | Awaiting approver sign-off |
| `Approved` | Ready for payment (follows configured approval policy) |
| `Paid` | Payment processed |

<Note>
  Bills created via API are automatically approved and enter the workflow at the **Approved** status.
</Note>

### Draft Bills

Draft bill endpoints are available to create and update draft bills through the API. Draft bills must be approved through the Ramp dashboard — approval via API is not supported.

## Creating a Bill

Use `POST /developer/v1/bills` to create a bill. The request requires several UUIDs from other endpoints:

| Endpoint | Field | Used as |
|---|---|---|
| `GET /entities` | `id` | `entity_id` |
| `GET /entities` | `bank_accounts[].id` where `usage_type = BILL_PAY_BANK_ACCOUNT` | `source_bank_account_id` |
| `GET /vendors` | `id` | `vendor_id` |
| `GET /vendors/{id}/contacts` | `id` | `vendor_contact_id` |
| `GET /vendors/{id}/accounts` | `id` | `vendor_account_id` (ACH/wire/SWIFT only) |

<Warning>
  If you don't see a bank account with `usage_type = BILL_PAY_BANK_ACCOUNT` in your `GET /entities` response, Bill Pay hasn't been set up for your organization. Follow the Bill Pay setup guide in the Ramp Help Center, then retry.
</Warning>

### Uploading Attachments

Upload invoice PDFs or supporting documents using `POST /developer/v1/bills/{bill_id}/attachments`. This endpoint accepts `multipart/form-data`.

## Payment Methods

### Pay by Ramp Card

Paying by Ramp Card earns cashback on bill payments.

**Existing Ramp Card** — Use already-created cards. Manually share card details with vendors, then use transaction matching to link the payment once it clears.

**Single-Use Virtual Card** — Ramp generates a unique virtual card for each bill, with the limit set to the bill amount.
- *Card delivery disabled:* Card details available once the bill is approved; manually share with vendors
- *Card delivery enabled:* Ramp automatically emails card details to the vendor's primary contact on the due date, with reminders at 2, 4, and 7 days

Single-use cards auto-lock when the spending limit is reached, after 14 days (delivery enabled) or 21 days (delivery disabled).

### Bank Transfer Methods

These methods require a `source_bank_account_id` from `GET /entities`:

| Method | Requirements | Processing Time |
|---|---|---|
| ACH | Verified vendor bank account | 2–3 business days |
| Check | Vendor address on file | 5–7 business days |
| Wire | Verified vendor bank account | Same-day (domestic) |
| SWIFT | Verified vendor bank account | Varies (international) |

<Note>
  Payments arrive by `payment_arrival_date` (cannot be a weekend or bank holiday). Expedited ACH and Wire payments are not available via API.
</Note>

## Webhooks

Use webhooks to receive real-time notifications when bills are created, approved, rejected, or paid. See the [Webhooks guide](/guides/webhooks) for setup instructions.

## Accounting Sync

Bills sync automatically with direct accounting connections (QuickBooks, NetSuite, etc.). For custom integrations, see the [Accounting guide](/guides/accounting).

Query with `sync_ready=true` to find bills ready for export.

<Warning>
  Bills cannot be synced via API with Universal CSV. Use the dashboard export instead.
</Warning>

## Three-Way Match (Bill → PO → Item Receipt)

To verify a bill matches its purchase order and item receipt:

1. Get the bill → extract `purchase_order_ids`, `line_items[].purchase_order_line_item_id`, and `inventory_line_items[].purchase_order_line_item_id`
2. Get the purchase order → extract `item_receipt_ids`
3. Get the item receipt → confirm `line_items[].purchase_order_line_item_id` matches the bill

When all three documents share the same line item IDs, you have a verified match.

## API Limitations

- You cannot trigger or manage bill approvals through the API
- Bills created via API are automatically approved
- Batch payments (single payment for multiple bills) are not supported
- OCR upload for draft bills is not supported via API
- Managing entity bank accounts is not supported via API
